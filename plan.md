# Toy Dairy — 三人 60 小时协作计划

> 目标：在 **60 人时**（3 人 × 约 20h，或 3–4 天密集冲刺）内交付 **Web 可演示 MVP**。  
> 技术路径：**Web 优先**（跨 OS 协作）→ 后期再打包为桌面 App（Electron / Tauri）。  
> 产品依据：见 [`readme.md`](./readme.md)（PRD 原文）。

---

## 1. 60 小时能交付什么（MVP 边界）

### 必须做完（演示闭环）

| 模块 | 用户能做什么 |
|------|----------------|
| 玩偶身份卡 | 新建玩偶：名称 / 出生日期 / 出生地 / 性格关键词 → AI 补全星座、简介、独白 → 生成身份卡 |
| 上传记录 | 相册选图 / 纯文字；填日期、地点、类型（旅行/日常） |
| AI 日记 | 按玩偶性格生成**第一视角**日记文案 |
| 时间轴 | 按时间列出该玩偶全部日志，可点进详情 |
| 基础「我的」 | 玩偶列表切换、简单账号（或本地演示账号） |

### 明确不做（留给下一阶段）

- 激活端 / 硬件
- 玩偶社交、社区动态
- 完整旅行地图打卡（可先用城市列表代替）
- 纪念日回忆展厅 + 幻灯片 + 拍立得分享（可做入口占位）
- 实时 AI 对话、插画风格迁移
- 会员 / 支付 / 小红书微信深度联动
- 桌面 App 打包（仅在文档预留方案）

### 技术选型（Web 优先，便于三人不同 OS）

| 层 | 推荐 | 说明 |
|----|------|------|
| 前端 | React + TypeScript + Vite | 跨平台开发；后期可嵌进 Electron/Tauri |
| UI | Tailwind + 少量组件库（如 shadcn） | 快速出温暖手帐风界面 |
| 后端 | Node.js + Fastify/Express **或** Supabase | 若想省运维优先 Supabase（Auth + DB + Storage） |
| 数据库 | PostgreSQL（Supabase）或 SQLite 本地演示 | MVP 以「能存能读」为准 |
| 对象存储 | Supabase Storage / 本地 `uploads/` | 照片 |
| AI | OpenAI 兼容 API（文案 + 可选多模态识图） | 身份卡补全、日记生成；Key 只放服务端 |
| 部署 | 前端 Vercel / Cloudflare Pages；API Railway / Render / 本机演示 | 黑客松可用本机 + ngrok |
| 桌面（后续） | Tauri 2 或 Electron 包 Web | **不计入本 60h** |

## 2. 三人角色与职责

| 代号 | 角色 | 核心职责 | 主要产出 |
|------|------|----------|----------|
| **A · 前端** | Web UI / 交互 | 页面、路由、组件、状态、调用 API | 可点的 Web 应用壳 + 主流程页面 |
| **B · 后端** | API / 数据 / 存储 | 数据模型、CRUD、鉴权、上传、对接 AI 服务封装 | REST/JSON API + DB Schema |
| **C · AI & 集成** | Prompt / 联调 / 体验 | Prompt 与生成质量、元数据、端到端串联、部署与演示脚本 | AI 服务、联调通过的主路径、Demo 清单 |

### 协作原则

1. **接口契约优先**：Hour 0–4 三人一起锁 OpenAPI/接口表，再分头写。  
2. **主干不断**：每天至少 1 次「创建玩偶 → 上传 → 出日记 → 时间轴可见」冒烟。  
3. **一个人阻塞不堵全队**：AI 未就绪时前端用 Mock；后端未就绪时前端用 MSW/假数据。  
4. **Key 与隐私**：API Key 仅服务端；用户图默认不训练、不进公开仓。  
5. **PR 小步**：功能分支 + 短 PR；`main` 始终可演示。

## 3. 接口契约（Hour 0 锁定，改动需三人同意）

最小资源：

```
POST   /auth/demo                 # 可选：演示登录 / 匿名 session
GET    /toys                      # 当前用户玩偶列表
POST   /toys                      # 创建玩偶（可先不带 AI，再 PATCH 补全）
GET    /toys/:id
PATCH  /toys/:id                  # 更新人设
POST   /toys/:id/generate-profile # AI 生成星座/简介/独白

POST   /toys/:id/entries          # 创建日志（multipart: 图 + 字段）
GET    /toys/:id/entries          # 时间轴列表
GET    /entries/:id               # 详情（含 AI 文案）
POST   /entries/:id/regenerate    # 重生成日记文案
```

**创建玩偶请求示例**

```json
{
  "name": "Luna",
  "birthDate": "2026-07-23",
  "birthPlace": "上海迪士尼",
  "role": "旅行搭子",
  "traits": ["温柔", "胆小", "好奇"]
}
```

**日志 Entry 字段**

- `type`: `travel | daily | memorial | text`
- `date`, `location`, `title`, `userNote`, `mood`
- `imageUrl`（可选）
- `aiDiary`（服务端写入）

## 4. 时间线（60 人时）

按 **3 人并行 × 约 20h/人** 规划。墙钟可用 3 天（每天约 6–7h）或 1 个长周末。

### Phase 0 · 对齐（0–4h，共同）

| 事项 | 负责人 |
|------|--------|
| 再读 PRD，确认 MVP 清单与「不做」列表 | 全员 |
| 锁技术栈、仓库结构、分支规范 | 全员 |
| 画 5 个主页面线框（身份卡 / 新建玩偶 / 加号编辑 / 时间轴 / 详情） | A 主导，C 辅助 |
| 写接口契约 + 数据表草案 | B 主导 |
| 准备 AI Key、Prompt 草稿、示例输入输出 | C 主导 |
| 建 monorepo 或 `web/` + `server/` 空项目可跑通 | A+B |

**验收**：`npm run dev` 前后端能起；Swagger/接口 md 已合并。

---

### Phase 1 · 垂直切片（4–20h）

目标：一条**假数据也能走通的竖切**，再换真实现。

| 人 | 任务 |
|----|------|
| **A** | 路由壳 + 底部 Tab（日志 / 成长占位 / + / 玩偶 / 我的）；新建玩偶表单 UI；身份卡展示组件 |
| **B** | `toys` / `entries` 表；创建/列表 API；本地或云存储上传接口（可先 base64/本地盘） |
| **C** | `generate-profile`、`generate-diary` 两个函数（先 CLI/单测）；固定 JSON schema 输出；失败重试与兜底文案 |

**验收**：用 curl/Postman 能建玩偶并拿到 AI 简介；前端能展示 Mock 身份卡。

---

### Phase 2 · 主路径接通（20–40h）

| 人 | 任务 |
|----|------|
| **A** | 编辑记录页（图/文、日期地点）；时间轴列表 + 详情页；玩偶切换；加载/错误/空状态 |
| **B** | 上传鉴权与关联玩偶；列表筛选（按玩偶/时间）；Entry 与 AI 字段落库；CORS/部署配置 |
| **C** | 接入真实上传链路：有图时可选「场景描述」进 Prompt；日记风格贴合性格；联调 A/B；写 3 套演示用玩偶种子数据 |

**验收**：**真机浏览器**完成：建玩偶 → 上传一张旅行图 → 生成日记 → 时间轴可点开。

---

### Phase 3 · 打磨与演示（40–60h）

| 人 | 任务 |
|----|------|
| **A** | 视觉统一（手帐/温暖风）、身份卡分享图（可选 html2canvas）、移动端宽度适配 |
| **B** | 稳定性：校验、限流简单版、日志、备份演示数据脚本 |
| **C** | 端到端演示脚本、README 运行说明补充到 plan 外文档、录屏、已知问题列表；预研桌面打包 1 页说明（不实现） |

**验收**：任意一人按脚本 5 分钟演示无阻塞；准备好 2 个故事线（迪士尼出生 / 鼓浪屿日落）。

## 5. 按角色的详细任务清单

### A · 前端（约 20h）

- [ ] Vite + React + TS 工程与目录约定  
- [ ] 布局：底部导航 + 中间「+」  
- [ ] 页面：玩偶主页（身份卡）、新建/编辑玩偶、编辑记录、日志列表、日志详情、我的（简版）  
- [ ] API client 与类型（与 B 契约同步）  
- [ ] 乐观 UI / Toast / 空状态  
- [ ] 响应式：优先手机宽度 390，兼顾桌面浏览器  

**不做**：复杂动画、完整地图、纪念日展厅动效（可用静态卡片占位）。

### B · 后端（约 20h）

- [ ] 项目脚手架 + 环境变量模板 `.env.example`  
- [ ] Schema：`users`（可简化）、`toys`、`entries`  
- [ ] CRUD + 文件上传  
- [ ] 内部调用 C 的 AI 模块（HTTP 或同进程 service）  
- [ ] 基础错误码与校验  
- [ ] 部署一键脚本或 Docker（可选）  

**不做**：复杂权限矩阵、支付、多租户计费。

### C · AI 与集成（约 20h）

- [ ] Prompt v1：身份卡（星座规则 + 性格介绍 + 一句话独白）  
- [ ] Prompt v1：第一视角日记（日期地点 + 性格 + 用户备注/图片描述）  
- [ ] 输出强制 JSON，便于落库  
- [ ] 无 Key / 超时 / 内容安全时的 fallback  
- [ ] 与 B 集成；与 A 对字段  
- [ ] 演示数据 + 演示话术  
- [ ] 《后续桌面化》半页：Tauri 包 `web` dist 的迁移注意点  

**不做**：微调模型、多角色社交、长对话 memory 系统。

## 8. 演示检查表（Deadline 前必过）

1. 新建玩偶「小熊 Luna」，出生地上海迪士尼，性格温柔胆小好奇 → 看到身份卡与独白  
2. 上传鼓浪屿相关图或纯文字 → 生成第一视角日记  
3. 时间轴出现该条，详情页文案可读、不崩  
4. 切换第二只玩偶，数据不串  
5. 刷新页面数据仍在  
6. 演示网络差或 AI 失败时有友好提示/兜底文案

## 9. 风险与对策

| 风险 | 对策 |
|------|------|
| AI 不稳定/贵 | 缓存同输入结果；演示用预生成；fallback 模板句 |
| 上传跨域/体积 | 限 5MB；服务端压缩；本地演示盘路径 |
| 接口对不齐 | 契约文件进 Git；改接口必须 PR 通知全员 |
| 60h 不够 | 砍「成长档案页」为只读统计数字；砍重生成与分享图 |
| OS 不一致 | 统一 Node 版本（`.nvmrc`）；只用 Web；禁本机强依赖路径 |

## 10. 60h 之后（不占用本次工时）

1. 成长档案页 + 城市列表/简易地图  
2. 纪念日卡片 + 拍立得分享图  
3. 照片 → 插画  
4. 玩偶 AI 对话（带旅行记忆检索）  
5. **Web → 桌面 App**（Tauri/Electron 套壳 + 本地文件系统可选）  
6. 激活端 / 社交增量

## 11. 一句话分工

> **A 做出能点的壳与主流程页面，B 管住数据与上传，C 让玩偶「会说话」并保证三人拼起来能 5 分钟演示。** 一切以 PRD 中的身份卡 + 旅行日记 + 时间轴闭环为北极星，60 小时内不做第二曲线。
